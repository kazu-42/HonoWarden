# v0.1.0-alpha Release Notes

Draft status: feature-freeze materials prepared on 2026-07-06.

HonoWarden is pre-alpha and must not be used to store real secrets.

## Scope

- package and runtime metadata version `0.1.0-alpha`
- API-only encrypted vault sync server for Cloudflare Workers
- Hono runtime with D1 relational state and R2 object-storage binding
- restricted account bootstrap
- prelogin KDF discovery
- password and refresh token grants
- refresh-token rotation and reuse invalidation
- authenticated empty and item sync
- owner-scoped custom equivalent-domain metadata in `/api/sync`,
  `/api/domains`, and `/api/settings/domains`
- custom equivalent-domain update through `POST`/`PUT /api/settings/domains`
- account profile read and display-name mutation
- folder CRUD
- login and secure-note cipher create/update/lifecycle
- direct folder and cipher read APIs with bounded list pagination
- cipher-scoped encrypted attachment upload, download, delete, and sync
  metadata backed by R2
- revision conflict protection
- device revoke
- `GET /api/devices`
- `GET /api/devices/identifier/:identifier`
- anonymous `GET /api/devices/knowndevice` preflight using encoded email and device headers
- revoke all other sessions with recent password authentication
- encrypted device key update routes for active owner-scoped devices
- login defenses
- TOTP setup and login challenge flow
- recent password authentication for TOTP setup routes
- TOTP setup reuse guard for already-enabled accounts
- authenticated TOTP disable route with recent password authentication
- TOTP disable deletes retained setup secret and replay state
- authenticated TOTP change route with recent password authentication
- TOTP change keeps the active secret enabled while pending replacement verify
  is outstanding
- TOTP change start and verify replay guards
- recent password-authenticated `POST /api/accounts/export` user export API
- `totp.change` audit event coverage
- `totp.disable` audit event coverage
- `backup.export` audit event coverage
- folder, cipher, and attachment vault mutation audit event coverage
- opt-in global request quota buckets with secret-safe abuse reporting
- backup/restore CLI secret-safe audit packets
- D1 audit-event persistence with 365-day bounded retention cleanup
- bounded retention cleanup for transient auth-defense rows
- explicit unsupported-feature responses for organization and public
  file-sharing surfaces
- operator backup/restore CLI wrapper
- account disable/enable operator CLI wrapper
- opt-in secret-safe audit events
- scheduled encrypted remote backup workflow and remote backup evidence
- compatibility fixture matrix
- user isolation regression coverage
- synthetic two-user dogfood and disabled-user lifecycle evidence packet
- security review materials
- inbound inquiry Email Worker handler for metadata-only D1 ingestion of
  allowed `honowarden.com` contact mailboxes
- inquiry inbox schema for threads, messages, and events with raw body and
  attachment storage disabled by default

## Not Included

- Web Vault, hosted vault UI, or browser session surface
- public registration
- Organizations or shared vaults
- organization membership, roles, shared collections, or collection assignment
- organization policy mutation or enforcement
- organization-scoped equivalent-domain rules or global equivalent-domain
  catalog management
- collection create, update, delete, or cipher assignment behavior
- Send or public file-sharing
- Emergency Access or delegated recovery
- account lifecycle admin UI and production lifecycle execution evidence
- bulk trusted-device approval and login-with-device workflows
- live official-client attachment upload/download/delete evidence
- live official-client user export evidence
- independent security audit
- broad live client matrix promotion beyond the recorded CLI and browser
  extension smokes
- live_regression promotion without a ready live regression packet and redacted
  evidence
- inquiry mailbox UI, AI triage, approved outbound replies, raw MIME storage,
  attachment storage, and Linear issue creation automation

## Compatibility

Compatibility remains conservative. The CLI row is promoted to `live_smoke`
after synthetic login, sync, item lifecycle, official CLI one-step TOTP login,
refresh grant, and HTTP recent-auth lifecycle evidence. The browser extension
row is promoted to `live_smoke` after a synthetic self-hosted login, initial
sync, account profile reads, and empty-vault render. Desktop, Android, and iOS
rows stay at `fixture_only` until live client runs are captured with synthetic
data and redacted evidence. Browser-extension, desktop, Android, and iOS TOTP
UX remain unproven. `live_regression` remains unclaimed until
`pnpm live:regression:packet -- --strict` records login, sync, item lifecycle,
refresh, session revoke, and selected auth lifecycle coverage.

## Known Risks

- no independent security audit
- remote production backup and local fresh-target restore evidence is committed,
  but no remote disposable Cloudflare restore drill has been recorded yet
- audit logging remains disabled by default until the target environment has
  migration `0007` applied and operator access approved
- user export has no dedicated global quota beyond bearer auth, recent-auth,
  password-grant login defense, and platform limits
- Worker runtime Logpush to R2 is configured, but downstream SIEM/vendor
  alerting and automated log-retention deletion are not productized
- TOTP wrapping-secret rotation lacks migration tooling
- access-token signing uses a symmetric secret without key id rotation support
- Cloudflare account-level access controls are documented, but least-privilege
  token remediation and break-glass rotation are still pending
- inquiry inbox receives only metadata by default; raw body and attachment
  retention/deletion paths must be separately enabled and evidenced before
  storing real report content

## Release Gate

Do not tag `v0.1.0-alpha` until:

- CI passes on the release commit
- migration freeze hashes match
- dependency audit evidence is current
- release docs are reviewed
- staging dry-run evidence is recorded
- backup fresh-target restore drill is recorded
- scheduled remote backup workflow and remote backup evidence are recorded
- synthetic client login/sync evidence is recorded
- synthetic two-user dogfood and disabled-user denial evidence is recorded
- `pnpm release:tag:preflight -- --strict --check-remote` reports `ready` on
  the clean release commit
- [Alpha Tagging Runbook](tagging-runbook.md) has been followed with explicit
  operator approval for tag creation and push
- `pnpm release:github:plan -- --strict --check-remote` reports `ready` after
  the tag has been pushed and verified
