# Packet 03: Docs And Workflow

## Objective

Document the bundle in the release runbook and workflow evidence.

## Files

- `docs/release/tagging-runbook.md`
- `docs/current-state.md`
- `.workflow/week-26-release-evidence-bundle/*`

## Do

- Require the bundle before tag approval.
- Keep tag and release write commands approval-gated.
- Document implemented and intentionally unimplemented actions.

## Do Not

- Claim that a generated bundle is itself approval to tag.

## Expected Output

Operators have one command to capture pre-tag evidence before copying the
approval text.
